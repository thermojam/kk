## ЧАСТЬ 1 · TELEGRAM-ИНТЕГРАЦИЯ (ядро v3)

### Решение по таргету

Основной поток ведёт в **личку** `https://t.me/xenia_kamensky`.
Личка не поддерживает `?start=` (это только для ботов) — поэтому:

1. **Трекинг источника на стороне сайта** — каждая кнопка перед открытием TG отправляет цель в Яндекс.Метрику (`reachGoal`).
2. **Контекст источника зашит в текст сообщения** — предзаполненный `?text=` служит и приветствием, и меткой, откуда пришёл человек. Ксения по тексту видит источник.

Предзаполненный текст — это приглашение: пользователь может его отредактировать перед отправкой. Для лички это норма и плюс (не выглядит автоматом).

**Два типа TG-ссылок на сайте:**

| Тип | Где встречается | Реализация | reachGoal | `?text=` |
|---|---|---|---|---|
| Tracked CTA (в личку) | Хедер, Hero, все 5 карточек Услуг, CTA-баннер под Услугами, Футер · TG-контакт | `TelegramButton` | ✅ | ✅ |
| Plain link (публичный канал) | Футер · «Канал отзывов» (`t.me/kmensky_case`) | `<a target="_blank" rel="noopener noreferrer">` | ❌ | ❌ |

Воронка считается только по tracked-CTA. Канал отзывов — публичная ссылка вне воронки, без приветственного текста и без цели Метрики.

### Единая утилита

```typescript
// lib/telegram.ts
const TG_USERNAME = 'xenia_kamensky';

export function tgLink(text: string): string {
  return `https://t.me/${TG_USERNAME}?text=${encodeURIComponent(text)}`;
}
```

```
lib/analytics/metrika.ts
- YM_ID в env
- Скрипт Метрики вставляется ТОЛЬКО после opt-in в cookie-баннере
```

**Cookie-баннер** (`components/legal/CookieBanner.tsx`):

```
Сайт использует cookie и Яндекс.Метрику для обезличенной
статистики посещений (IP, страницы, время визита).
Оператор данных — ООО «Яндекс». Срок хранения — 12 месяцев.
Подробнее: Политика cookie · Политика ПДн.

[ Отказаться ]          [ Принять ]
```

Текст покрывает обязательный минимум 152-ФЗ (цели, состав, оператор, срок) — окончательная формулировка согласовывается с юристом. До «Принять» — скрипт Метрики не вставляется. Выбор хранится в localStorage в виде `{version, decision, timestamp}`, где `version === POLICY_VERSION`; при смене версии баннер показывается повторно. TTL 12 месяцев. На мобайле баннер закреплён внизу.

## ЧАСТЬ 4 · ЮРИДИЧЕСКИЙ СЛОЙ (сокращён)

### Что нужно (минимум под v3)

| Документ | Нужен? | Обоснование |
|---|---|---|
| Политика cookie | ✅ Да | Метрика ставит cookie — обязательно, opt-in |
| Политика обработки ПДн | ✅ Да (короткая) | Метрика передаёт обезличенные данные Яндексу (оператор). 152-ФЗ серая зона — безопаснее иметь. Адаптировать под деятельность Ксении |
| Согласие `pdn-general` | ❌ Нет | Нет формы — нечего согласовывать |
| Согласие `pdn-health` | ❌ Нет | Скрининг вручную в TG, не на сайте |
| Договор-оферта | ❌ Нет на сайте | Оплата вне сайта; при желании Ксения присылает в переписке |

### Требования к юрстраницам (важно по запросу)

Обе страницы (`/privacy`, `/cookies`):
- **Адаптированы под деятельность эксперта** — формулировки про психолога/женские практики, реквизиты ИП, упоминание Яндекс.Метрики как единственного сборщика данных, отсутствие форм сбора ПДн на сайте.
- **Стилизованы под лендинг** — те же токены (typography, neutral-цвета, контейнер 1200px), хедер-лого и футер. Не «голый текст», а часть сайта.
- **Якорные ссылки футера ведут на главную** — на юр.страницах все пункты футер-навигации используют формат `/#anchor` (например, `/#about`), не локальные `#about`. Иначе клик «Обо мне» из футера `/privacy` не сработает.
