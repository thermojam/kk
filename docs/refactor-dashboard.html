Рекомендации за пределами этого рефакторинга

P1 Visual regression testing
Сейчас «без визуальных изменений» — обещание на честном слове. Установить Playwright + screenshot-сравнение для главной страницы (desktop + mobile) — это разблокирует более смелые UI-рефакторинги. Стоимость: ~1-2 часа на инфраструктуру, далее CI запускает автоматически.
P1 Выделить <Section>-обёртку
7 секций используют одинаковую разметку <section className="container-page py-16 lg:py-24">. Выделить компонент Section с props {id, className?, children} и заменить копии. Это даст единую точку для будущих изменений spacing-системы. Риск низкий, экономия — ~1-2 строки на секцию + единая семантика. Не сделал в этом проходе: CLAUDE.md §2 — «не выделять абстракции, если не просили»; пользователь не подтверждал такое выделение явно.
P1 Footer: «мёртвые» ссылки href="#"
Три ссылки в нижней панели футера (Политика ПДн, Оферта, Cookie) ведут на # — нерабочие. По SPEC они обязательны (юридический минимум). Либо завести страницы /legal/privacy, /legal/offer, /legal/cookies, либо временно убрать ссылки до момента появления контента.
P1 Open Graph / Twitter metadata
В app/layout.tsx нет блока metadata.openGraph и metadata.twitter. При шеринге в мессенджерах / соцсетях карточка будет пустой. Добавить картинку public/og.png 1200×630 и заполнить title/description/image.
P1 sitemap.xml + robots.txt
Для SEO single-page лендинга нужен минимальный sitemap и robots. Next 16 поддерживает их через файлы app/sitemap.ts и app/robots.ts. Реализация — ~10 строк каждый.
P2 Дизайн-токены для теней и duotone-tint
Оставшиеся 4 rgba() в коде — design-shadows карточек (WorkAreaCard, CaseCard) и purple-tint фона логотипа. Их стоит вынести в CSS-переменные (--shadow-card-soft, --shadow-card-glow, --color-primary-500-tint8) или использовать color-mix() для tint. Это даст single source of truth для shadow-системы.
P2 Bundle analyzer
Static bundle сейчас 1.46 MB. @next/bundle-analyzer покажет, что весит больше всего (вероятно — framer-motion ~80KB и embla-carousel). После анализа — оценить, нужны ли все эти зависимости или их можно lazy-load (next/dynamic).
P2 Lighthouse / Web Vitals аудит
Запустить npx lighthouse или Chrome DevTools → Lighthouse в продакшен-сборке. Метрики LCP/CLS/INP. Hero-картинка уже с priority и sizes — LCP должен быть хорош. CLS — проверить, что нет layout-shift при загрузке шрифтов (использовать next/font если ещё не).
P2 Контраст текста на тёмном фоне
text-white/[0.78] на bg-neutral-900 в Hero — на грани WCAG AA для body. Проверить через npm i -D @axe-core/cli или вручную через https://webaim.org/resources/contrastchecker/. Если ниже — поднять до 0.85 (визуальная разница незаметна).
P2 Унифицировать fadeUp-анимацию
Helper fadeUp(delay) определён локально в Hero.tsx. Похожая анимация — кандидат и для других секций (About, Services карточки). Если такая нужна — вынести в lib/motion.ts. Не делать «на будущее» — только если реально появится 2-й/3-й кейс использования.
P2 Заменить useLayoutEffect-хаки Embla на нативный API
CarouselArrows.tsx и CarouselDots.tsx используют useLayoutEffect + eslint-disable react-hooks/set-state-in-effect. Это legitimate (Embla — imperative API), но можно посмотреть, нет ли в Embla React-bindings нового хука useEmblaCarousel().selectedSnap или подобного — он мог бы избавить от ручной синхронизации.
P2 Skip-link «к содержимому»
Для клавиатурной навигации полезен скрытый <a href="#main">Перейти к содержимому</a>, который становится видимым при :focus. Сейчас при Tab с самого верха пользователь сначала проходит по всем пунктам шапки. Добавляется одна строка в app/(public)/layout.tsx.
