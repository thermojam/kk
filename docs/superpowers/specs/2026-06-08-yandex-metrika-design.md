# Дизайн: включение Яндекс.Метрики (счётчик 109730343)

**Дата:** 2026-06-08
**Статус:** утверждён к имплементации
**Связано:** интеграция аналитики в рамках MVP

## Контекст

В репозитории уже реализована полная клиентская интеграция Яндекс.Метрики с opt-in:

- `lib/analytics/metrika.ts` — IIFE-сниппет ЯМ; `initMetrika()` вызывается **только** после согласия; есть очередь `pendingGoals` и API `reachGoal(name)`.
- `lib/analytics/consent.ts` — хранение решения в `localStorage` (`kk.consent.v1`) с TTL 12 месяцев и привязкой к `POLICY_VERSION`.
- `components/legal/CookieBanner.tsx` — баннер «Принять / Отказаться»; при `accepted` дёргает `initMetrika()`.
- `app/(public)/layout.tsx` — рендерит `<CookieBanner />`.
- `content/legal/privacy.ts`, `content/legal/cookies.ts` — описывают использование ЯМ и opt-in поток.

**Чего не хватает:**

1. `NEXT_PUBLIC_YM_ID` нигде не заполнен → в проде счётчик не активируется (`metrika.ts:21` `if (!YM_ID) return`).
2. Webvisor сейчас `false` (`metrika.ts:48`) — решено перевести в `true`. Это меняет условия обработки и требует синхронной правки юр.слоя и инкремента версии политик.

## Решения

| Вопрос | Решение |
|---|---|
| Где хранить ID `109730343` | В `.env.production`, закоммитить в git. ID публичный, не секрет. |
| Набор фич сниппета | `clickmap + trackLinks + accurateTrackBounce + webvisor` |
| Кастомные цели (`reachGoal`) | Не подключаем сейчас. Только pageview. API в коде остаётся, но не вызывается. Клик-цели — в кабинете ЯМ через CSS-селектор (out-of-scope). |
| `<noscript>` pixel-фолбэк | **Не добавляем.** Противоречит opt-in: сработал бы до согласия. |
| Поведение при пустом `NEXT_PUBLIC_YM_ID` | Текущая логика `metrika.ts:21` — early return. Сохраняется. |
| `POLICY_VERSION` после правки политик | `1` → `2`. Это автоматически переспросит согласие у всех, кто принял v1 (см. `consent.ts:21`). |

## Manual verification (по правилам проекта — без автотестов)

1. `npm run lint && npx tsc --noEmit` — без ошибок.
2. Чистая сборка с прод-окружением:
   ```
   npm run build && npm start
   ```
   Открыть `http://localhost:3000/` в инкогнито, DevTools → Network, фильтр `mc.yandex.ru`:
   - При загрузке: **баннер виден**, в Network **нет** запросов к `mc.yandex.ru`.
   - Нажать «Отказаться» → баннер исчез, в Network по-прежнему **нет** ЯМ.
   - `localStorage.clear()` в консоли → перезагрузить → баннер снова, нажать «Принять» → в Network появляется `tag.js`, затем beacon-визит.
3. Эмуляция миграции `v1 → v2`:
   ```js
   localStorage.setItem('kk.consent.v1', JSON.stringify({
     version: 1, decision: 'accepted', timestamp: Date.now()
   }));
   ```
   Перезагрузить → **баннер должен показаться**, ЯМ не должен грузиться до повторного «Принять».
4. На страницах `/cookies/` и `/privacy/` — в тексте видно упоминание Webvisor; дата обновлена на 2026-06-08.
