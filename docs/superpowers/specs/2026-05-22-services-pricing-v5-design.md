# Services · Pricing V5 (Serif Italic Display Number) — design

Цена в карточке услуги (`#services`) переводится в формат «крупная серифная курсивная цифра + мелкая sans-приписка». Cormorant Italic — тот же шрифт, что в hero-display и подзаголовках услуг, поэтому решение бренд-консистентно. Источник: визуальный мокап `services-pricing-variants.html`, вариант V5 (одобрен пользователем 2026-05-22).

## Цели

- Цены перестают быть «мелким серым списком» — становятся визуальным якорем нижней части карточки.
- Структура цены явно описана в типах: значение (крупно) отделено от контекста (мелко).
- На featured-карточке (фиолетовый фон) цена окрашена в `accent-500` (контраст высокий, правило «accent — не текст на белом» соблюдено).
- Free-консультация по-прежнему рендерится без блока цен; CTA прижата к низу.

## Не-цели

- Менять `Services.tsx`, `ServicesCarousel.tsx`, `EmblaCarousel.tsx`, бэйджи, дисклеймер, CTA-кнопку, выравнивание карточек по высоте, мобильную карусель.
- Менять секцию «Под каруселью CTA-баннер».
- Добавлять новые токены палитры / радиусов / типографики.
- Парсить старые строки цены — модель данных реструктурируется (вариант A из брейншторма).

## Модель данных

```ts
// content/home.ts

export type ServicePrice = {
    /** Крупная серифная часть. Примеры: "5 000 ₽", "3 / 6". */
    value: string;
    /** Мелкая sans-приписка справа от value. Примеры: "за 1,5 часа", "месяцев сопровождения". Опционально. */
    meta?: string;
};

export type Service = {
    id: string;
    badge: string;
    title: string;
    subtitle?: string;
    description: string;
    /** Список тарифов (0..N). Пустой массив — блок цен не рендерится (free consult). */
    prices: ServicePrice[];
    /** Дополнительная строка курсивом под тарифами. Например, «стоимость — после диагностической встречи». */
    pricingNote?: string;
    cta: { label: string; href: string };
    featured?: boolean;
    disclaimer?: string;
};
```

Старое поле `pricing: string[]` удаляется. `ServicePrice` экспортируется как top-level тип — может пригодиться для тестов / переиспользования.

## `ServiceCard.tsx` — изменения

Заменяется только блок `<ul className="...">` (текущие строки 55–64). Всё остальное (бэйдж, title, subtitle, description, disclaimer, CTA, обёртка `<article>`) — без изменений.

Логика:
- Если `prices.length === 0` и `!pricingNote` — блок цен не рендерится; `mt-auto` переходит на CTA-кнопку, чтобы CTA по-прежнему прижималась к низу.
- Иначе — внешний обёртывающий `<div>` с `mt-auto flex flex-col gap-2`, верхней границей `border-t` (`neutral-100` на default, `white/20` на featured) и `pt-3.5`. Внутри:
  - `<ul className="flex flex-col gap-2">` — список тарифов (по `<li>` на каждый), рендерится только если `prices.length > 0`. Каждый `<li>`: `flex flex-wrap items-baseline gap-x-3 gap-y-1`. `value` — Cormorant italic 500, `text-[32px] lg:text-[36px] leading-none tracking-[-0.01em]`, `text-primary-500` (default) / `text-accent-500` (featured). `meta` (если есть) — sans 13px, `text-neutral-500` (default) / `text-white/85` (featured).
  - `<p>` для `pricingNote` (если есть) — Cormorant italic 13px, `text-neutral-500` / `text-white/85`, рендерится последним в обёртке, после `<ul>`.

Заметки по реализации:
- `mt-auto` сейчас стоит на `<ul>`. После перехода: если pricing-блок рендерится — `mt-auto` на нём, иначе — на `<Button>` (через условный класс).
- Disclaimer (`<DisclaimerToggle>`) остаётся между pricing-блоком и кнопкой — место не меняется.
- Шрифт берётся из утилиты `font-serif` (которая уже маппится на Cormorant через next/font в globals.css). Стиль italic + жирность через inline classes.

Минимальная высота блока цены не фиксируется на компонент-уровне: серифная цифра занимает ~36–40px по высоте, чего достаточно, чтобы выравнивание карточек по `align-items: stretch` Embla работало без скачков. Если визуально появятся неровности — поправим в плане.

## SPEC.md §5 — изменения

1. В подразделах «Карточка А/Б/В/Г» заменить ASCII-представление цены на новый формат (например, для А: `5 000 ₽    за 1,5 часа` вместо `1,5 часа · 5 000 ₽`). Карточка Д не меняется (без цен).
2. В подразделе «Выравнивание контента карточки» строка про Pricing (`min-h-[3rem]` + `mt-auto` — прижимает прайс... — рендерится всегда (даже при пустом массиве)) переписывается:
   - блок цен рендерится только при непустом `prices` или наличии `pricingNote`;
   - `mt-auto` переходит на блок цен, либо на кнопку (если блока цен нет);
   - визуальная характеристика: серифная курсивная цифра 32/36px + sans-приписка 13px.

## Edge cases

- **`prices = []`, `pricingNote` отсутствует** (free-consult): pricing-блок не рендерится. CTA получает `mt-auto`.
- **`prices = []`, `pricingNote` присутствует**: внешняя обёртка с `border-t` и `pt-3.5` рендерится, внутри только `<p>` с заметкой (без `<ul>`). На сегодня такого кейса нет, но поведение определено.
- **Длинный `meta`**: переносится по `flex-wrap` под `value`. Высота карточки регулируется существующим `stretch`-механизмом.
- **Длинный `value`** (не ожидается, но): `value` не имеет жёсткого `max-width`; `tracking-[-0.01em]` и `leading-none` минимизируют риск переноса. Если столкнёмся в реальном контенте — добавим `whitespace-nowrap` локально.
- **Featured + accent-500**: фон карточки `primary-500`, контраст `accent-500` к `primary-500` — высокий, читаемо. Текущее правило палитры (accent не используется как текст на белом) не нарушается.
- **Reduced motion / a11y**: блок цен — статичный текст, без анимаций. Семантика: `<dl>` рассматривался, но `value`/`meta` — это не «термин / определение», а два визуальных компонента одного смыслового элемента; оставляем `<div>` со стандартным flow. Структуру цен `<ul><li>` сохраняем для скринридеров — каждый `ServicePrice` — это `<li>`.
