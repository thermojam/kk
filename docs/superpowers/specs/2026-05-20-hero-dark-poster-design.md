# Hero Dark Poster — design

Редизайн секции 1 (Hero) лендинга Ксении Каменской. Светлый постер с лиловым градиентом заменяется на «тёмный постер» во всех вьюпортах. Контент SPEC §1 (заголовок, подзаголовок, тексты CTA) не меняется.

## Цели

- Сменить визуальный регистр первого экрана с «мягкого мистического» на «постерный, премиальный».
- Не вводить новые токены палитры и не менять контент.
- Сохранить контракт компонента `Hero` (props/имя/якорь) и поведение анимации (fade-up, hover-scale, reduced-motion).
- Соответствие WCAG AA по контрасту во всех элементах.

## Не-цели

- Редизайн остальных секций (About, WorkAreas, Cases, Services, FAQ, Contact). Они остаются в светлой палитре. Резкий переход hero→about на тёмный→светлый трактуется как фича «киношного» захода.
- Изменение текстов заголовка/подзаголовка/CTA.
- Введение тёмной темы сайта в целом — это локальный тёмный hero, не глобальный dark-mode.

## Композиция

### Desktop (≥ 1024px)

- Высота секции: `min-height: 720px` (как сейчас в SPEC §1).
- Grid 2 колонки, асимметрия `1.15fr / 0.85fr`, gap 48px, container `max-width: 1200px` через `.container-page`.
- Левая колонка (текст):
  - eyebrow «Ksenia Kamenskaya» — Nunito 700, 11px, uppercase, `letter-spacing: 0.4em`, цвет `--color-accent-500`, margin-bottom 22px;
  - заголовок — Cormorant Italic 400, 72px, line-height 0.98, цвет white; завершающий фрагмент «и проявленности.» (точка включена) — оборачивается в `<span class="accent">` с цветом `--color-accent-500`. Полная разбивка: `Психология женского тела <span class="accent">и проявленности.</span>` — точный перенос строк подбирается на этапе реализации;
  - подзаголовок — Nunito 400, 16px, line-height 1.6, цвет `rgba(255,255,255,0.78)`, max-width 460px;
  - CTA-стек: primary (`Button variant="primary"`) → ghost (`Button variant="ghost"`), gap 14–18px, alignment `flex-start`.
- Правая колонка (портрет):
  - размер 300×380px (`aspect-ratio: 3 / 4`);
  - `border-radius: var(--radius-sm)` (6px);
  - фильтрация — duotone (см. ниже);
  - декоративная offset-рамка 1px `--color-accent-500` со смещением `-14px / +14px` справа-сверху.

### Mobile (< 1024px)

- Тёмный фон сохраняется. Это единый постер на всех экранах.
- Single column, padding `py-16`, gap 24px.
- Порядок: eyebrow → заголовок (Cormorant Italic, 40px) → подзаголовок (Nunito, 15px) → портрет (width `min(70vw, 320px)`, aspect 3/4, тот же duotone, без offset-рамки, выравнивание по центру) → CTA-стек, `align-items: flex-start`.
- Высота — auto (без `min-height`).

## Дуотон портрета

Подход — без SVG-фильтров, через стек CSS:

1. На `next/image` контейнере применяется фильтр `filter: grayscale(1) contrast(1.1)`.
2. Поверх контейнера — два абсолютно-позиционированных слоя с `mix-blend-mode`:
   - слой shadows: `background: var(--color-primary-500); mix-blend-mode: multiply;`
   - слой highlights: `background: var(--color-accent-500); mix-blend-mode: lighten;`
3. На стадии реализации проверяем визуально: если лицо «теряется» — fallback на лёгкую десатурацию (`filter: contrast(1.05) saturate(0.6) brightness(0.95)` без blend-слоёв, как было в моке V5).

Стартовая реализация — duotone (вариант 1+2). Переход на fallback (вариант 1 без blend-слоёв) — только если на визуальной проверке лицо «теряется» или цветовое решение бьёт с фотопрессетом 8.2.

## Декор и анимация

`HeroBackground.tsx` переписывается:

- Базовый фон: solid `--color-neutral-900`.
- Два radial-glow через CSS-градиенты:
  - нижний-левый — `radial-gradient(circle, var(--color-primary-500) 0%, transparent 70%)`, opacity 0.55, размер 460×460px, смещение `left: -160px; bottom: -180px`;
  - верхний-правый — `radial-gradient(circle, var(--color-primary-400) 0%, transparent 70%)`, opacity 0.35, размер 320×320px, смещение `right: 35%; top: -120px`.
- Лёгкое движение glow через Framer Motion:
  - нижний-левый glow: `animate: x [-15, 15, -15], y [-10, 10, -10]; duration 18s; ease easeInOut; repeat Infinity`;
  - верхний-правый glow — в противофазе: `animate: x [15, -15, 15], y [10, -10, 10]; duration 22s; ease easeInOut; repeat Infinity`.
- Частицы `--color-accent-500`: те же 5 позиций, через существующий CSS `@keyframes hero-particle` и класс `.hero-particle`. Никакой логики не меняется.
- Видимость декора:
  - lg+ — glow и частицы активны;
  - < lg — `HeroBackground` возвращает `null` (статичный фон без декора, как у текущей мобильной версии);
  - `prefers-reduced-motion: reduce` — `HeroBackground` возвращает `null`.

Входные анимации Hero-контента (`Hero.tsx`):

- fade-up наследуется 1:1: заголовок delay 0, подзаголовок 0.1s, CTA 0.2s, портрет 0.3s. Reduced motion → статика (как сейчас).
- Hover на портрете: `scale(1.02)`, duration 0.4s, easeOut — сохраняется.

## Правки SPEC.md §1

Меняются следующие участки (строки 38–50 в текущей версии SPEC.md):

1. **«Анимация (концепт «параллакс на портрете»)»** — переименовать концепт в «Dark Poster», переписать перечень: фон neutral-900 + два radial-glow primary-500/400 + частицы accent-500.
2. **Адаптив** — заменить «портрет справа, текст слева, высота 720px» и «портрет сверху 50vw, текст снизу» на новые формулировки с указанием тёмного фона на всех вьюпортах и асимметричного grid 1.15fr/0.85fr на lg+.
3. **Дополнительный пункт «Цвет и обработка»**: фон neutral-900, primary CTA — accent-500 на neutral-900, портрет — duotone purple/orange (fallback: лёгкая десатурация).

Контент-блок (заголовок, подзаголовок, тексты CTA, фото-источник) **не меняется**.

## Риски и открытые точки

- **Duotone на конкретном фото.** Лицо в портрете обработано пресетом брендбука 8.2; mix-blend двух фирменных цветов может либо красиво «лечь», либо «съесть» полутона. Решение: проверить на этапе реализации, при необходимости — fallback на десатурацию.
- **Резкий стык Hero→About.** Чёрный hero → светлый About — это смелое решение. Если на ревью внешний вид «бьёт по глазам», добавим тонкий переходный градиент 80–120px в нижней части Hero (`linear-gradient(to bottom, transparent, var(--color-neutral-0))`). Решение откладываем до визуальной проверки.
- **Серифный заголовок 72px на тёмном.** Cormorant Italic на чёрном при ширине шрифта 400 может выглядеть тонко. Если по факту «не звучит» — поднимаем weight до 500 (требует подгрузки соответствующего веса шрифта через next/font).
