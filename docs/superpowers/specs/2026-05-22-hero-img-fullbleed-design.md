# Hero img Full-Bleed — design

Уточнение визуального решения секции Hero (после Hero Dark Poster, 2026-05-20). Фон оригинального портрета удалён вручную пользователем; новое размещение — фото на всю высоту секции, нижняя кромка вплотную к нижнему краю hero. Layout — вариант B (Full-Bleed Right) из брейншторма 2026-05-22.

## Цели

- Сделать портрет визуальным «героем» секции — крупным, доминирующим, как на постере.
- Прижать нижнюю кромку фото к нижнему краю секции на всех вьюпортах.
- Сохранить контракт компонента `Hero` (props/имя/якорь) и поведение fade-up анимации.
- Использовать тот факт, что у фото теперь прозрачный фон — без декоративной рамки и без скруглений.

## Не-цели

- Менять текст заголовка, подзаголовка, CTA. Контент SPEC §1 остаётся 1:1.
- Менять HeroBackground (radial-glow + частицы) — он не трогается.
- Применять цветовые фильтры к портрету (дуотон, ч/б, десатурация) — фото остаётся в натуральных цветах.
- Менять offset-рамку как декор — она удаляется целиком (без замены).

## Композиция

### Desktop (≥ 1024px)

- Секция: `min-height: 720px`, тёмный фон neutral-900, HeroBackground (glow + частицы) — без изменений.
- Grid-контейнер (`.container-page`):
  - 2 колонки `1.15fr / 0.85fr`, gap 48px.
  - Padding меняется: `lg:py-24` → `lg:pt-24 lg:pb-0`. Нижний padding обнуляется, иначе фото не доходит до пола секции.
  - `lg:items-center` сохраняется как дефолт для grid.
- Левая колонка (текст): `lg:self-center`. Тексты, eyebrow, CTA, размеры — без изменений.
- Правая колонка (портрет):
  - `lg:self-end` — прижата к нижнему краю grid (= нижнему краю секции, так как `pb-0`).
  - Контейнер фото: `position: relative`, высота `lg:h-180` (720px = `min-height` секции), ширина — авто по aspect фото (фигура «определяет» ширину).
  - `<Image>` с явными `width`/`height` (натуральные размеры файла) и `style={{ height: '100%', width: 'auto' }}`, выровнено по нижнему краю. Нижняя кромка фото = нижний край секции. Если ширина фигуры (~540px при aspect 3:4) превышает колонку 0.85fr (~390px), излишек выходит вправо, секция `overflow-hidden` обрезает горизонтальный спил.
  - **Без фильтров.** Без duotone-слоёв.
  - **Без offset-рамки и без `border-radius`.**
- Hover-эффект: `scale(1.01)` на контейнере фото, duration 0.4s, easeOut. Reduced motion → без hover.

### Mobile (< 1024px)

- Single column: текст → фото снизу.
- Грид → одна колонка (наследуется из дефолтного grid без `lg:grid-cols-[…]`).
- Padding секции на мобайле: `py-16` → `pt-16 pb-0`. Тот же приём: фото касается пола.
- Правая колонка (теперь — нижний flex-item): занимает всю ширину секции, без max-width.
- Фото: `width: 100%`, `height: auto`, `object-position: bottom center`. Aspect определяется самим изображением.
- HeroBackground остаётся `null` на мобайле (как сейчас).

## Что удаляется из `Hero.tsx`

- `<span aria-hidden>` — декоративная offset-рамка (сейчас lines 78–90).
- `borderRadius: var(--radius-sm)` со стилей контейнера фото.
- Фиксированные размеры контейнера `lg:w-75 lg:h-95` и `aspect-3/4`. Контейнер на мобайле также теряет `w-[min(70vw,320px)]` — заменяется на full-width.
- Класс `relative overflow-hidden w-full h-full` у внутреннего div фото — заменяется на простой `relative w-full h-full` (overflow-hidden теперь не нужен, без border-radius нечего обрезать).

`motion.div` с fade-up + whileHover остаётся (hover-scale меняется с `1.02` на `1.01`).

## Правки SPEC.md §1

Меняются два участка:

1. **Адаптив (десктоп):** «портрет 300×380, offset-рамка accent-500, border-radius 6px» → «портрет на всю высоту секции (720px), full-bleed в правой колонке, без offset-рамки и без border-radius».
2. **Адаптив (мобайл):** «портрет под текстом, max-width min(70vw, 320px), aspect 3/4» → «портрет под текстом, на всю ширину секции, height auto, нижняя кромка вплотную к низу секции».
3. **Цвет и обработка:** убрать упоминание duotone purple/orange и fallback десатурации. Заменить на «портрет без цветовых фильтров, фон удалён, силуэт в натуральных цветах на neutral-900».

Контент-блок (тексты заголовка, подзаголовка, CTA, источник фото) **не меняется**.

## Цвета и контраст (WCAG)

Все текстовые элементы остаются на `neutral-900` фоне с прежними соотношениями (≥ 8:1 по заголовку, ≥ 14:1 по подзаголовку, ≥ 8:1 по eyebrow и primary CTA). Фигура портрета не перекрывает текстовую зону (отдельная колонка grid на десктопе; отдельный блок ниже текста на мобайле).

## Риски и открытые точки

- **Aspect портрета не задокументирован.** Если оригинал hero.png близок к 3:4 — на десктопе ширина фигуры выйдет около 540px, а правая колонка grid (~390px) узковата. На full-bleed фигура подгоняется по высоте контейнера и по ширине будет шире колонки — это нормально, secция уже `overflow-hidden`. Если визуально «прижимается» — поднимаем `lg:h-180` или меняем grid-пропорции (1fr / 1fr).
- **Mobile + длинное фото.** Если у изображения вытянутый портрет (4:3 или хуже), на мобайле блок может «отъесть» половину экрана. Решение откладываем на визуальную проверку. План B: добавить `max-height: 60vh` на мобайле.
- **Текст на мобайле близко к фото.** Сейчас gap 24px между текстом и фото; если визуально слишком плотно — увеличить gap до 32px.

## Acceptance

- Фото без фона целиком видно от макушки до ступней на десктопе и мобайле.
- Нижняя кромка фото ровно по нижнему краю секции (px-точно, без зазора и без перекрытия `About`).
- Текст слева/сверху не пересекается с фигурой и не теряет контраст.
- Hover на десктопе даёт `scale(1.01)`; reduced-motion отключает hover и fade-up.
- HeroBackground поведение не изменилось.
- SPEC.md §1 обновлён согласно «Правки SPEC.md §1».
- Lint и build проходят без новых ошибок (`npm run lint`, `npm run build`).
