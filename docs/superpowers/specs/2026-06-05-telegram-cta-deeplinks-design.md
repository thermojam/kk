# Telegram CTA Deeplinks Design

## Цель

Все CTA-кнопки, ведущие в личный Telegram Ксении, должны открывать чат с
предзаполненной смысловой заготовкой, соответствующей намерению пользователя.

Контактная ссылка Telegram в футере и ссылка на канал отзывов остаются обычными
ссылками без предзаполненного текста.

## Текущее состояние

- `TelegramButton` уже принимает `goal` и `text`.
- CTA услуг, Hero и баннера после услуг уже передают отдельные тексты.
- `tgLink()` сейчас игнорирует переданный текст и возвращает обычную ссылку на
  Telegram.
- Финальный блок `Offer` передаёт пустой текст.
- Telegram-контакт в футере не использует `TelegramButton` и не входит в scope.

## Решение

Сохранить существующий контракт `TelegramButton` и восстановить формирование
публичной Telegram-ссылки с параметром `text`:

```ts
export function tgLink(text: string): string {
    return `${CONTACTS.telegram}?text=${encodeURIComponent(text)}`;
}
```

Каждая CTA-кнопка продолжает передавать собственные `goal` и `text`.
Существующая отправка целей в Яндекс.Метрику не меняется.

## Изменения

- `lib/telegram.ts`: `tgLink(text)` добавляет URL-кодированный параметр `text`.
- `components/sections/ContactCtaBanner.tsx`: обновляется текст общей CTA.
- `components/sections/Offer.tsx`: пустой текст заменяется смысловой заготовкой.
- `content/home.ts`: у «Силы Берегини» убирается фраза о готовности заполнить
  анкету, чтобы CTA не заявляла действие от имени пользователя.

## Вне Scope

- Telegram-контакт `@xenia_kamensky` в футере.
- Канал отзывов.
- Email и внутренние ссылки.
- Тексты на видимых кнопках.
- Список и названия целей Яндекс.Метрики.

## Проверка

1. `npm run lint` проходит без ошибок.
2. `npm run build` проходит без ошибок.
3. Каждая из восьми CTA-ссылок содержит `?text=` и корректно URL-кодирует свою
   заготовку.
4. Telegram-контакт и канал отзывов в футере не содержат `?text=`.
5. Каждая CTA сохраняет существующую цель Яндекс.Метрики.
