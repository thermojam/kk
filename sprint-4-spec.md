**Проект:** Лендинг Ксении Каменской · психология женского тела
**Стек:** Next.js 16 (App Router, RSC), TypeScript, Tailwind CSS
**Источник истины по дизайну:** Brand Book v2.0 + согласованный прототип `kamenskaya-v2.html`
**Дата:** июнь 2026

---

## 0. Принцип спринта

Прототип `kamenskaya-v2.html` — это **визуальный контракт** (раскладка, отступы, поведение, цвета). Он **не источник ассетов**. Всё, что в прототипе сделано «на коленке», в проекте переделывается на продакшен-уровень. Явный список того, что НЕ переносим как есть — в разделе 7. К деталям относимся строго: пиксель, токен, контраст.

## 1. Архитектура и структура

### 1.1 Дерево (App Router)
```
app/
  layout.tsx            // шрифты, метаданные, <html lang="ru">
  page.tsx              // сборка секций (RSC)
  globals.css           // Tailwind + CSS-переменные (токены брендбука)
components/
  sections/
    Hero.tsx            // client (анимации появления)
    About.tsx           // RSC
    WorkWith.tsx        // client (слайдер на мобайле)
    Stories.tsx         // client (слайдер на мобайле)
    Services.tsx        // client (карусель + стрелки + точки)
    Faq.tsx             // client (аккордеон)
    Offer.tsx           // RSC
    Footer.tsx          // RSC
  ui/
    Button.tsx          // primary / secondary, типизирован
    Logo.tsx            // финальный SVG-знак (см. 7.1)
    Icon.tsx            // набор SVG-иконок (см. 7.2)
    Carousel.tsx        // переиспользуемый scroll-snap враппер
  layout/
    FixedLogo.tsx       // client (scroll-логика подложки)
lib/
  constants.ts          // ссылки TG, контакты, реквизиты
  content.ts            // тексты секций (услуги, истории, FAQ, квалификации)
public/
  images/               // оптимизированные фото (см. 7.3)
  logo/                 // SVG-знак в вариантах
```

### 1.2 RSC vs Client
- **RSC по умолчанию.** `'use client'` только там, где есть интерактив или scroll-API: Hero (анимации), Services/Stories/WorkWith (слайдер), Faq (аккордеон), FixedLogo.
- Контент (тексты, цены, истории, FAQ) выносится в `lib/content.ts` как типизированные структуры — секции мапят данные, без хардкода в разметке.

### 1.3 Принцип данных
```ts
// пример типизации, lib/content.ts
type Service = {
  id: string;
  badge: string;
  featured?: boolean;
  title: string;
  quote: string;
  description: string;
  price: { value: string; note: string };
  cta: { label: string; href: string };
};
```
Все секции с повторяющимися карточками (Services, Stories, WorkWith, FAQ) рендерятся из массивов. Никаких трёх скопированных блоков в JSX.

| Токен | HEX | Назначение |
|---|---|---|
| primary-50 | #F7F1FC | фон бейджей, иконок |
| primary-300 | #C9A3E8 | номера историй, бордеры стрелок |

| ink-950 | #0F0F16 | фон футера |
| n100/300/500/700 | по брендбуку | бордеры, вторичный текст |

### 2.2 Критические правила цвета (Brand Book 5.3 — НЕ нарушать)
- **Accent (#FFA552) НИКОГДА не текст на светлом фоне** (контраст 2.1:1). На кнопке Accent текст всегда **Ink (#1E1E2E)**, не белый.
- Accent как текст — только на тёмном (hero, оффер, футер).
- Пропорция 60-30-10: нейтральный фон доминирует, accent ≤10%.

### 2.3 Типографика
- **Nunito** (300–900) — весь UI и body.
- **Cormorant Garamond Italic** (400–600) — только акценты: hero h1, заголовок оффера, цитаты услуг, номера историй. Правило брендбука: **не более одного крупного блока на экран**.
- Подключение через `next/font/google` (самохостинг, не CDN-link). Брендбук гл. 12: «шрифты самохостинговые — для скорости и приватности». `display: 'swap'`.
- Шкала (десктоп/мобайл): Display 72/48, H1 48/32, H2 40/30, H3 24/20, Body 16/15.

## 3. Секции — поведение и acceptance

### 3.1 FixedLogo
- `position: fixed`, top/left по брейкпоинтам (desktop 28/64, mobile 16/20).
- При `scrollY > 100` — появляется подложка (белая 92%, blur, тень).
- **Открытый вопрос для проекта:** подложка vs смена цвета знака через IntersectionObserver. На спринт берём **подложку** (дешевле). Зафиксировать в Logo.tsx возможность темы (light/dark знак) на будущее.
- ✅ Логотип читается на всех секциях (тёмный hero, белые секции, тёмный футер).

### 3.2 Hero (client)
- Градиент строго из токенов: `linear-gradient(120deg, primary-900, primary-700 55%, #1a0833)` + radial фиолет слева + radial accent за фигурой. **Без мутной середины.**
- Заголовок: один блок, `max-width: 11ch`, line-height 1.08, left-align. «и проявленности.» — accent, в потоке.
- Две кнопки: Primary → TG (прямая ссылка, без start-параметра), Secondary → плавный скролл к `#services`.
- Фото справа, прижато к низу, `object-fit: contain`, drop-shadow.
- Анимация появления: staggered (eyebrow → title → sub → cta), `animation-delay`. На `prefers-reduced-motion` — отключить.
- ✅ Контраст текста на градиенте ≥ 4.5:1 (проверить hero-sub, сейчас #D8CFE6).

### 3.3 About (RSC)
- Грид 380px / 1fr, фото 3:4. Квалификации — 2 колонки, мапятся из массива.
- Мобайл: колонка, фото max-width 320px, квалификации в 1 колонку.

### 3.4 WorkWith / Stories / Services — единый слайдер
- **Десктоп:** grid 3 колонки.
- **Мобайл (≤860px):** общий компонент `Carousel.tsx` — `scroll-snap-type: x mandatory`, карточка `flex: 0 0 85vw`, `scroll-snap-align: center`, скрытый скроллбар.
- **Stories:** акцент-полоса = `::before` внутри `overflow:hidden` карточки (по всей высоте, углы по радиусу). НЕ border-left.
- **Services:** карточка — flex-колонка, `.desc { flex:1 }`, `.foot { margin-top:auto }` → цена+CTA всегда на одной линии у всех карточек независимо от объёма текста. Featured-карточка (фиолет, белый текст, accent-бейдж).
- **Services стрелки/точки:** на десктопе стрелки активны (скролл карусели), точки — индикатор. На мобайле стрелки скрыты, свайп + точки.
- ✅ Контент не «прыгает»: добавление абзаца в любую карточку услуг не сдвигает низ.

### 3.5 FAQ (client)
- Аккордеон, первый открыт по умолчанию. Анимация max-height.
- A11y: `<button aria-expanded>`, `aria-controls`, управление с клавиатуры.

### 3.6 Offer (RSC) — новый блок после FAQ
- Фиолетовый градиент + тёплое свечение. Заголовок Cormorant Italic. CTA → TG.
- Финальный конверсионный экран перед футером.

### 3.7 Footer (RSC)
- Фон ink-950 (плотный, не мутный).
- Контакты: иконка + текст (Telegram / Email / Канал отзывов). Hover заливает иконку accent.
- Обязательно (Brand Book 11): дисклеймер, реквизиты ИП (ОГРНИП, ИНН), ссылки на Политику ПДн и Cookie.

## 5. Производительность и качество (Brand Book гл. 12)

- Lighthouse Performance ≥ 90, Accessibility ≥ 95.
- `next/image` для всех фото (не base64-инлайн как в прототипе). Форматы AVIF/WebP, правильные `sizes`, `priority` для hero.

- Семантика: один `<h1>`, корректная иерархия заголовков, `alt` у всех изображений, `lang="ru"`.
- Метатеги: OG-image, twitter:card.

## 6. Юридический минимум на старте (Brand Book гл. 11)

Не дизайн, но блокирует прод — учесть в архитектуре сразу, не «потом»:
- Хостинг и любые формы/email-база — **в РФ-зоне** (242-ФЗ). Статику можно где угодно, ПДн — только РФ.
- Cookie-баннер (российский подход: одна кнопка «Принять» + ссылка на политику, до начала трекинга).
- Аналитика — Яндекс.Метрика (не GA4).
- Footer: ссылки Политика ПДн / Cookie / реквизиты — уже в макете, должны вести на реальные страницы.
- Формулировки: «психолог», «консультирование» — да; «терапия/лечение/диагностика» — нет.

*На этом спринте форм нет (CTA ведут в TG), поэтому 152-ФЗ по формам не триггерится. Появятся формы — отдельная проработка.*

## 7. Что переделать с нуля (НЕ переносить из прототипа)

Прототип импровизировал ассеты. В проекте — строго:

### 7.1 Логотип
- В прототипе — **моя реконструкция SVG** по описанию из брендбука (две зеркальные К, круг, точка). **Это НЕ финальный знак.**
- В проект: взять **оригинальный SVG-знак** из брендбука (гл. 4). Параметры: viewBox 0 0 120 120, толщина линий 4.5px, радиус точки 4px, скругления round, минимум 24×24px.
- Варианты: основной (на тёмном/светлом), инверсия, только знак (favicon/аватар). Favicon в 16/32/180/512.
- Охранное поле: высота знака × 0.5.

### 7.2 Иконки
- В прототипе — **символы-заглушки** (✈ ✉ ★ 💗 🧭 ✨). **Не продакшен.**
- В проект: единый набор **SVG-иконок** (один стиль, один stroke-width). Telegram, email, звезда/канал — для футера. Иконки категорий «С чем работаю» — подобрать консистентный сет (Lucide или кастом в стиле брендбука). Точка из знака может работать буллетом (разрешено брендбуком).

### 7.3 Изображения
- В прототипе hero-фото — **автоматическая вырезка фона** (возможен ореол по краю волос). В проект: **профессиональная вырезка** (ручная/нейросеть с чистым краем), экспорт PNG с альфой + WebP.
- Все фото: единый пресет цветокоррекции (Brand Book гл. 8 — тёплый, плёночный, приглушённый). Прогнать hero и about через пресет.
- Подготовить ретина-версии (@1x/@2x), отдать через `next/image`.

### 7.4 Контент
- Тексты в прототипе — с согласованных скриншотов. Финально вычитать на ToV (гл. 3): «ты», без «вылечу/гарантирую», цитаты клиентов обезличены и с разрешения.
- Цены — вилками, как в брендбуке (гл. 7), не единой цифрой, где это уместно.
